# GRO Insights Dashboard — Complete Implementation Spec (canonical)

> **Source.** Authored in Claude Design and pasted in verbatim 2026-06-27 as the standard for
> GroLabs insights surfaces. The dashboard is dark on `#0c0c0f`, with translucent frosted "subtle"
> cards; charts are hand-built SVG (no chart library) generated by a small script at load and
> animated in.

## Corrections & integration notes (read before implementing)

1. **Single-donut renderer fix (verified).** §6's `single` donut draws a full `<circle>` with
   inline `stroke-dasharray="frac (1−frac)"`, but §6's own CSS `.ring svg circle.seg { stroke-dasharray:1 }`
   **overrides** that → it renders a complete ring, not the fraction. Fix: render the single donut as
   a **partial arc `<path>`** like the split segments, so the path length *is* the fraction and the
   draw-on animation works:
   ```js
   // in renderSVGs(), the single branch:
   svg.appendChild(el("path",{ d:arc(cx,cy,r,0,Math.min(359.9,360*frac)),
     fill:"none", stroke:roleColor(svg.dataset.ra), "stroke-width":sw,
     "stroke-linecap":"round", class:"seg", pathLength:"1" }));
   ```
   (Alternatively, stop forcing `stroke-dasharray:1` on `circle.seg` in CSS.) Verified in the mock.
2. **Token reconciliation with `design-tokens.md`.** This spec introduces a *new* surface treatment
   (frosted `rgba(255,255,255,0.05)` + `backdrop-filter`, `#0c0c0f` canvas, mono labels, the `--n1..n5`
   neutral ramp, semantic vs brand modes) that extends — and in places differs from — the current
   `design-tokens.md` (`--gl-bg #131316`, borderless-by-no-fill). When this lands in `globals.css`,
   fold these tokens into the `--gl-*` dictionary deliberately (don't leave two parallel systems):
   the frosted `.tile` becomes a named GroLabs surface, `#0c0c0f` a canvas token, and the neutral
   ramp + semantic/brand modes get `--gl-*` names. Until then, this doc is the source for the
   dashboard surface.
3. **Mobile/responsive breakpoints are real (keep them).** The spec's `≤768px` blur→solid fallback
   and `≤1100/720px` column collapse are intentional. (The chat mock lowered them only to render
   multi-column at a 680px preview viewport — do **not** carry that change into the app.)

- **Page background:** `#0c0c0f`, optional faint glow `radial-gradient(110% 70% at 50% -10%, rgba(250,225,148,0.05), transparent 55%)`

### Color philosophy
Two modes; default is **Semantic**:
- **Semantic:** bright green/yellow/red reserved for *meaning* (good / watch / problem). Everything
  representational (compositions, magnitudes, category breakdowns) uses the neutral grey ramp
  `--n1…--n5` or the subtle device tints.
- **Brand:** decorative; yellow `--accent` used freely.
- If you implement one, implement **Semantic** (root `class="dash sem"`).

### Build order (for Claude Code)
1. `:root` tokens + fonts (§1–2).
2. Subtle-surface CSS on `.tile`/`.dash-head` (§3) + grid (§4).
3. Component CSS (§5–10, §14).
4. JS: `roleColor` (§11), helpers + `renderSVGs()` (§12, with the single-donut arc fix), reveal `play()` (§13).
5. Default root to `class="dash sem"` (Semantic).

Every measurement, color, easing, and SVG parameter is taken verbatim from the working file — do not
round or "tidy" them; the look depends on these exact values.
