Key Relationships
Provider ↔ Brand (many-to-many): One provider can distribute multiple brands. One brand can be sold by multiple providers. The ProviderBrands join table tracks which providers supply which brands.
Provider → PriceList (one-to-many): Each price list import is tied to exactly one provider. A provider can have many historical price lists.
PriceList → PriceListItem (one-to-many): Each price list contains many line items (one per product). Items link to the master Product catalog.
Brand → ProductVariant (one-to-many): Every product variant belongs to exactly one brand (manufacturer). The entity named "ProductVariant" represents the sellable SKU (e.g. "Royal Canin Adult 15kg" is a variant).
Category → ProductVariant (one-to-many): Variants are grouped into categories. Each category has target & min margin settings.
MAPRule (polymorphic): Rules can originate from a Brand (manufacturer-mandated MAP) OR a Provider (contractual terms). The source_type enum ('brand' or 'provider') + source_id FK determines the origin. Rules can apply to all variants from that source (variant_id null) or be variant-specific.
PriceBatch → PriceBatchItem (one-to-many): A batch is a worksheet of pending price changes. Each item is one variant's pricing calculation (costs, charm price, manual overrides, violations).
Polymorphic MAPRule Design
The MAPRule table uses source_type (enum: 'brand' | 'provider') + source_id (UUID) to point to either the Brand table or Provider table. This allows:

• Brand-level rules: "Royal Canin mandates all Prescription Diet variants must be ≥Q650" → source_type='brand', source_id=royal_canin_id, variant_id=null
• Provider-level rules: "Distributor XYZ negotiates max Q700 for Hill's c/d 8kg" → source_type='provider', source_id=xyz_id, variant_id=hills_cd_8kg_id

When validating a price, the system queries all active rules where:
1. variant_id matches OR variant_id is null (applies to all)
2. source_type='brand' AND source_id = variant's brand_id, OR
3. source_type='provider' AND source_id = any provider who has supplied that variant
